Je veux créer une application web mobile personnelle appelée provisoirement « ClimbTrack ». Elle doit être optimisée principalement pour iPhone et pouvoir être ajoutée à l’écran d’accueil comme une application.

L’application est uniquement destinée à mon usage personnel. Je ne veux pas de réseau social, de profil public, de partage entre utilisateurs, de classement, de publicité, d’abonnement ou de contenu communautaire.

DESIGN GÉNÉRAL

Créer une interface moderne, très propre, minimaliste et sportive.

Palette de couleurs :

* fond noir ;
* cartes gris très foncé ;
* séparations gris moyen ;
* textes principaux blancs ;
* textes secondaires gris clair ;
* boutons principaux blancs avec texte noir ;
* vert discret uniquement pour signaler un exercice terminé ;
* rouge discret uniquement pour les avertissements ou suppressions.

Ne pas utiliser de couleurs vives pour l’interface normale.

L’application doit être :

* parfaitement lisible sur iPhone ;
* utilisable facilement avec une seule main ;
* rapide ;
* fluide ;
* moderne ;
* entièrement en français ;
* compatible avec le mode portrait ;
* installable comme PWA sur l’écran d’accueil ;
* capable de fonctionner hors ligne après le premier chargement.

Les données doivent rester personnelles et être sauvegardées localement sur mon appareil. Prévoir également un bouton pour exporter toutes les données en fichier JSON et un bouton pour les réimporter.

SUSPENSIONS ET ASSISTANCE DES PIEDS

Pour toutes les suspensions sur réglette de 25 mm ou moins, prévoir un champ « Assistance ».

Choix :

* deux pieds au sol, appui fort ;
* deux pieds au sol, appui moyen ;
* deux pieds au sol, appui léger ;
* un pied au sol ;
* pointes des pieds ;
* sans assistance ;
* lesté.

Pour le moment, la valeur par défaut doit être « deux pieds au sol, appui moyen ».

Pour les suspensions de 25 mm, je peux parfois tenir brièvement avec très peu d’assistance. Je dois pouvoir enregistrer séparément un test sans aide, même s’il ne dure que quelques secondes.

Les graphiques doivent distinguer :

* temps avec assistance ;
* niveau d’assistance ;
* temps sans assistance ;
* charge ajoutée éventuelle.

Ne jamais comparer directement une suspension assistée avec une suspension sans aide comme s’il s’agissait de la même performance.

PROGRESSION AUTOMATIQUE

L’application doit analyser l’évolution, mais ne jamais changer les valeurs sans mon accord.

Après deux séances consécutives où toutes les séries prévues sont réussies, proposer une progression.

Afficher :

* Accepter ;
* Garder le même niveau ;
* Modifier manuellement.

Ne modifier qu’un paramètre à la fois.

Pour la musculation

Proposer généralement une petite augmentation :

* environ 1 kg pour les charges légères ;
* environ 2 à 2,5 kg pour les charges moyennes ;
* jamais augmenter en même temps la charge et les répétitions.

Pour les tractions lestées

Proposer généralement :

* +1 kg ;
* +1,25 kg ;
* ou +2,5 kg selon le matériel disponible.

Pour les suspensions

Ne pas diminuer automatiquement la taille de réglette.

Ordre de progression :

1. même réglette ;
2. même durée ;
3. réduire légèrement l’aide des pieds ;
4. passer de deux pieds à un pied ;
5. passer aux pointes des pieds ;
6. réaliser quelques secondes sans aide ;
7. augmenter progressivement le temps sans aide ;
8. seulement après maîtrise sans aide, proposer éventuellement du lest.

Ne pas proposer de lest tant que la suspension sans assistance n’est pas correctement maîtrisée.

Pour la pince disque

Lorsque 3 séries de 20 secondes sont réussies deux fois :

* proposer une petite augmentation de charge ;
* sinon proposer 25 secondes ;
* ne pas augmenter charge et durée en même temps.

Pour le gainage

Lorsque toutes les séries sont réussies deux fois :

* proposer +5 secondes par série ;
* lorsque la durée devient trop longue, proposer une variante plus difficile.

Plateau

Si aucune progression n’est visible pendant 4 à 6 séances, afficher :
« Progression stable depuis plusieurs séances »

Puis proposer une seule option :

* conserver la charge ;
* augmenter le repos ;
* réduire temporairement le volume ;
* tester une variante ;
* réaliser un test de record après récupération.
