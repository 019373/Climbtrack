Autre truc à ajouter : Ajoute une carte de chaleur musculaire intelligente sur le corps humain. Je ne veux pas un simple coloriage fixe, mais un système qui calcule automatiquement la sollicitation musculaire.
1. Couleurs

Utiliser cette échelle de couleurs :

⚫ Noir = 0 pt (muscle non travaillé)
⚪ Blanc = 1 à 9 pts
⚪ Gris = 10 à 19 pts
🟢 Vert = 20 à 34 pts
🟡 Jaune = 35 à 49 pts
🟠 Orange = 50 à 64 pts
🔴 Rouge = 65 à 79 pts
🟣 Violet = 80 pts et plus
La transition entre les couleurs doit être fluide.


2. Points de base

Chaque exercice doit posséder une valeur de base pour chaque muscle sollicité.

Exemple :

Avant-bras : 10 pts

Abdominaux : 1 pt
Ces valeurs sont uniquement des points de départ.


3. Les points doivent être intelligents

Les points ne sont pas fixes.

L’application doit automatiquement ajuster légèrement les points selon :

le temps de maintien
le poids ajouté
le niveau d’assistance
la taille de la réglette
les répétitions
le nombre de séries
Exemple :

Suspension 20 mm :

Version de base

→ Avant-bras = 10 pts

Même exercice :

sans assistance

→ environ 11 ou 12 pts

Même exercice :

15 secondes

→ environ 11 pts

Même exercice :

25 secondes

→ environ 13 pts

Même exercice :

avec poids

→ environ 14 ou 15 pts

Les changements doivent rester relativement faibles.

Une version plus difficile ne doit jamais doubler les points.

Je souhaite que les modifications restent progressives.


4. Accumulation

Les points doivent s’additionner sur toute la séance.

Exemple :

5 séries de suspension

●
3 séries de tractions lestées

●
3 séries de pince

=

Avant-bras : 67 pts

Le corps affichera donc les avant-bras en rouge.

Système de badges - ClimbTrack

Je souhaite ajouter une page Badges dans l’application.

Le but n’est pas de créer un jeu, mais de rendre la progression plus motivante et agréable à suivre.

Les badges doivent avoir un design premium, moderne et minimaliste, parfaitement intégré au style actuel de l’application.


Design

Chaque badge possède :

une illustration unique
un nom
une courte description
une date de déblocage
un niveau (Bronze, Argent, Or ou Diamant)
Tous les badges sont visibles dès le début.

Les badges non débloqués apparaissent entièrement gris.

Une fois débloqués, ils prennent leurs vraies couleurs.


Apparence

Je veux une forme hexagonale.

Le contour doit avoir un léger effet 3D (sans être trop chargé).

Le niveau est représenté uniquement par le contour.

Par exemple :

🥉 Bronze

contour bronze métallique
🥈 Argent

contour argent métallique
🥇 Or

contour or métallique
💎 Diamant

contour bleu cristal avec quelques reflets
Je veux quelque chose de discret.

Le contour peut être légèrement en relief avec quelques reflets, mais il ne doit pas ressembler à un jeu vidéo.

L’intérieur reste totalement plat (2D).

Le dessin est donc en 2D.

Seul le contour possède un très léger effet premium.


Badges vivants

Chaque badge doit représenter directement son objectif.

Je ne veux pas de simples icônes.

Je veux de vraies petites illustrations.

Exemples :

Suspension 20 mm

→ une vraie réglette avec “20 mm” gravé dessus.

Tractions lestées

→ une barre de traction avec un disque.

Pince

→ une main pinçant un disque.

Face pull

→ une poulie.

Développé militaire

→ une barre au-dessus de la tête.

Squat

→ une silhouette faisant un squat.

Couleurs

→ une vraie prise d’escalade de la couleur correspondante.

Chaque badge doit être immédiatement reconnaissable sans lire son nom.


Évolution

Je ne veux pas plusieurs badges pour le même exercice.

Je veux un seul badge qui évolue.

Bronze → Argent → Or → Diamant.

À chaque évolution :

le contour change
quelques détails de l’illustration peuvent être améliorés
les couleurs deviennent légèrement plus riches
Le dessin principal reste toujours le même.


Déblocage

Lorsqu’un badge est obtenu :

petite animation
vibration si disponible
message :
🎉 Nouveau badge débloqué !

avec le badge affiché.

Important

Les seuils doivent être facilement modifiables dans un seul fichier de configuration (par exemple badges.ts ou badges.json).

Je souhaite pouvoir ajuster les objectifs sans modifier le reste de l’application.

Les illustrations doivent être réalisées en SVG ou dans un format vectoriel afin qu’elles restent parfaitement nettes sur iPhone et qu’elles puissent évoluer facilement avec les différents niveaux de badges.
